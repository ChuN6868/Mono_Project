# 概要

## ログイン画面による認証

AppService→「設定」→「認証」を開くとIDプロバイダーの項目がある。ここをMicrosoftと設定するとMicrosoftのログイン画面を使用できる<br>

- その右側に表示されている「アプリ（クライアント）ID」とは何？？
  - 各組織で発行された、そのアプリのためだけに作成されたID
  - テナントIDの方は、アプリごとではなく組織そのもののIDのこと（Bicepで出てくるtenantIdとは別なので注意）

- 「認証の設定」の右側の「編集」ボタンで詳細の設定が見れる
  - FEは302でログイン画面にリダイレクトさせる（Webサイトに推奨）
    - ここで設定してあることにより、「アプリのドメイン」にアクセスすると、Microsoftのログイン画面に遷移する
    - →ログインするとCookieにトークンが入り、ユーザー名等を取得できるようになる
    - FEの開発者ツールのCookieのトークンを削除した場合
      - 画面をリロードすると、「アプリのドメイン」へのアクセス判定となり、ログイン画面に遷移
      - FE内で別の画面に遷移すると、「アプリのドメイン内での遷移」であり、すでにCookieのトークンは関係なしにFEにユーザー情報が保持されているため普通に遷移できる
  - BEは認証されていないものであったらリダイレクトではなくエラーを返せばいいので401を返す（APIならこれ推奨）

### その他のIDプロバイダーと実用的な選び方

シーン | おすすめ
-- | --
社内システム | Microsoft Entra ID 一択
個人開発・技術系 | GitHub が手軽
一般向けサービス | Google + Apple（カバー率が高い）
既存の認証基盤がある | OpenID Connect で接続

※複数プロバイダーの併用も可能（下記のように複数を同時に設定して、ユーザーに選ばせることもできます。）
```
ログイン画面
  ├── Microsoftでログイン
  ├── Googleでログイン
  └── GitHubでログイン
```

## IDプロバイダー①：Google認証

```
ユーザー
  → FE (Web App) にアクセス
  → 未ログインなら Google ログイン画面にリダイレクト
  → Google アカウントでログイン
  → FE に戻る（認証済み）
```

### 必要なもの

| 必要なもの | 取得元 | 料金 |
|-----------|--------|------|
| Google クライアントID | Google Cloud Console | 無料 |
| Google クライアントシークレット | Google Cloud Console | 無料 |
| App Service (Free F1) | Azure | 無料 |

### Step 1: Google Cloud Console で OAuth クライアントIDを作成

#### 1-1. Google Cloud Console にアクセス

- [Google Cloud Console](https://console.cloud.google.com/) にGoogleアカウントでログイン

#### 1-2. プロジェクトを作成

1. 画面上部のプロジェクト選択 → **「新しいプロジェクト」**
2. プロジェクト名を入力（例: `mono-project-dev`）
3. **「作成」** をクリック

#### 1-3. OAuth 同意画面を設定

1. 左メニュー → **「APIとサービス」** → **「OAuth 同意画面」**
2. User Type: **「外部」** を選択（Googleアカウントを持つ全ユーザーが対象）
3. 以下を入力:

| 項目 | 値 |
|------|-----|
| アプリ名 | 任意（例: `Mono Project`） |
| ユーザーサポートメール | 自分のメールアドレス |
| デベロッパーの連絡先情報 | 自分のメールアドレス |

4. スコープ: デフォルトのまま（email, profile, openid）
5. **「保存して次へ」**

#### 1-4. 認証情報（クライアントID）を作成

1. 左メニュー → **「APIとサービス」** → **「認証情報」**
2. **「認証情報を作成」** → **「OAuthクライアントID」**
3. 以下を入力:

| 項目 | 値 |
|------|-----|
| アプリケーションの種類 | **ウェブアプリケーション** |
| 名前 | 任意（例: `Mono Project Dev`） |
| 承認済みリダイレクトURI | `https://<WebApp名>.azurewebsites.net/.auth/login/google/callback` |

> **リダイレクトURI の例:**
> `https://app-bicep-lesson-dev-fe-hogehoge-20260314.azurewebsites.net/.auth/login/google/callback`
> ※ WebApp名は `dev.bicepparam` の `webAppNameFrontend` の値に合わせる

4. **「作成」** をクリック
5. 表示される **クライアントID** と **クライアントシークレット** をメモする

> **重要:** クライアントシークレットはこの画面でしか表示されない場合があるため、必ず控えておく

### Step 2: Bicep を修正

#### 2-1. web-app.bicep に Google 認証パラメータを追加

`infrastructure/bicep/modules/web-app.bicep` に以下のパラメータを追加:

```bicep
// デフォルト値をfalseにする設定
@description('Google認証を有効にするか')
param enableGoogleAuth bool = false

@description('Google OAuth クライアントID')
param googleClientId string = ''

@secure()
@description('Google OAuth クライアントシークレット')
param googleClientSecret string = ''
```

`infrastructure/bicep/modules/web-app.bicep` に以下のGoogle認証設定を追加:

```bicep
// Google認証設定（enableGoogleAuth が true の場合のみ作成）
resource authSettings 'Microsoft.Web/sites/config@2023-12-01' = if (enableGoogleAuth) {
  parent: webApp
  name: 'authsettingsV2'
  properties: {
    globalValidation: {
      requireAuthentication: true
      unauthenticatedClientAction: 'RedirectToLoginPage'
      redirectToProvider: 'google'
    }
    identityProviders: {
      google: {
        enabled: true
        registration: {
          clientId: googleClientId
          clientSecretSettingName: 'GOOGLE_CLIENT_SECRET'
        }
        validation: {
          allowedAudiences: [
            googleClientId
          ]
        }
      }
    }
    login: {
      tokenStore: {
        enabled: true
      }
    }
  }
}

// クライアントシークレットをアプリ設定に格納
resource appSettings 'Microsoft.Web/sites/config@2023-12-01' = if (enableGoogleAuth) {
  parent: webApp
  name: 'appsettings'
  properties: {
    GOOGLE_CLIENT_SECRET: googleClientSecret
  }
}
```

#### 2-2. main.bicep に Google 認証パラメータを追加

`infrastructure/bicep/main.bicep` に以下のパラメータを追加:

```bicep
@description('Google OAuth クライアントID')
param googleClientId string = ''
// ↑ Google Cloud Console で取得したクライアントID

@secure()
@description('Google OAuth クライアントシークレット')
param googleClientSecret string = ''
// ↑ Google認証のシークレットをデプロイ時にコマンドラインで渡す（コードに含めない）
```

FE用モジュール呼び出しに認証パラメータを追加:

```bicep
// FE用 Web App を作成するモジュールを呼び出し（Google認証付き）
module webAppFrontend 'modules/web-app.bicep' = {
  name: 'webAppFrontendDeployment'
  params: {
    ~~ 
    その他パラメータ
    ~~
    enableGoogleAuth: true // Google認証を有効にする
    googleClientId: googleClientId // Google認証パラメータから渡す（誰のアプリか、を識別するだけなので設定ファイルに書き込んでよい）
    googleClientSecret: googleClientSecret // Google認証パラメータから渡す（secure() でマスクされる。暗証番号のようなものなので設定ファイルに書き込まない）
  }
}
```

> BE用の `webAppBackend` は変更不要（`enableGoogleAuth` のデフォルトが `false` のため）

#### 2-3. dev.bicepparam にクライアントIDを追加

`infrastructure/bicep/environments/dev.bicepparam` に追加:

```
param googleClientId = '<Step 1 で取得したクライアントID>'
```

> **クライアントシークレットは bicepparam に書かない。**
> デプロイ時にコマンドラインで渡す（次のステップ参照）

### Step 3: デプロイ

```bash
// 事前にプレビュー
az deployment group create \
  --resource-group <リソースグループ名> \
  --template-file infrastructure/bicep/main.bicep \
  --parameters infrastructure/bicep/environments/dev.bicepparam \
  --parameters googleClientSecret='<クライアントシークレット>' \
  --what-if

// デプロイ
az deployment group create \
  --resource-group <リソースグループ名> \
  --template-file infrastructure/bicep/main.bicep \
  --parameters infrastructure/bicep/environments/dev.bicepparam \
  --parameters googleClientSecret='<Step 1 で取得したクライアントシークレット>'
```

> `@secure()` を付けたパラメータはコマンドラインで渡すことで、コードに秘密情報を含めずにデプロイできる

### Step 4: 動作確認

1. デプロイ完了後、FE の URL にアクセス:
   ```
   https://<webAppNameFrontend>.azurewebsites.net
   ```
2. Google のログイン画面にリダイレクトされる
3. Google アカウントでログイン
4. アプリに戻り、認証済みの状態でアクセスできる

#### 認証情報の確認

ログイン後、以下のエンドポイントで認証情報を確認できる:

```
https://<webAppNameFrontend>.azurewebsites.net/.auth/me
```

レスポンス例:
```json
{
  "clientPrincipal": {
    "identityProvider": "google",
    "userId": "1234567890",
    "userDetails": "user@gmail.com",
    "userRoles": ["authenticated", "anonymous"]
  }
}
```

### トラブルシューティング

| 症状 | 原因と対処 |
|------|-----------|
| リダイレクトURIエラー | Google Cloud Console のリダイレクトURIが Web App の URL と一致していない。末尾の `/.auth/login/google/callback` を含めて正確に設定する |
| 403 Forbidden | OAuth 同意画面が「テスト」モードのままで、テストユーザーに追加されていない。同意画面でユーザーを追加するか「公開」に変更する |
| ログイン後に白い画面 | Web App 自体にアプリがデプロイされていない。認証設定とは別にアプリのデプロイが必要 |
| クライアントシークレットが無効 | シークレットの有効期限が切れている。Google Cloud Console で再作成する |

### 補足: セキュリティに関する注意

- **クライアントシークレットは Git にコミットしない**（`dev.bicepparam` に直接書かない）
- 本番運用では **Azure Key Vault** にシークレットを格納し、Key Vault参照で取得するのが望ましい
- OAuth 同意画面を「公開」にすると誰でもログインできるようになるため、必要に応じてアクセス制限を検討する

- 設定ファイルにシークレットを書かなくても、パイプラインのコードに書き込んだらセキュリティ対策で意味がなくなる
  - パイプラインではシークレットをコード外に保管し、実行時に注入する仕組みを使います。

```
× パイプラインYAMLに直接書く（コードに書くのと同じ）
  --parameters googleClientSecret='GOCSPX-xxxxx'

○ CI/CDの「シークレット変数」機能を使う
  --parameters googleClientSecret=$(GOOGLE_CLIENT_SECRET)
                                   ↑ 実行時にCI/CD側から注入される
```

各CI/CDサービスのシークレット管理
サービス | 設定場所 | コードに含まれるか
-- | -- | --
GitHub Actions | Settings → Secrets and variables → Actions | No
Azure DevOps | Pipelines → Library → Variable groups | No
Azure Key Vault | Azure Portal → Key Vault → シークレット | No
